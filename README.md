## Midterm Examination Reflection

1. What is Git used for?
Git is a version control system that allows developers to track changes in their code, collaborate with others, and manage different versions of their projects efficiently. It can serve as a backup system, facilitate collaboration, and help maintain a history of changes for better project management.

2. What is the difference between Git and GitHub?
Git is a distributed version control system that allows developers to manage and track changes in their code locally. GitHub, on the other hand, is a web-based platform that hosts Git repositories and provides additional features such as collaboration tools, issue tracking, and project management. While Git is the tool for version control, GitHub is a service that leverages Git for online collaboration and sharing. Git can also be used independently of GitHub, while GitHub requires Git to function.

3. Why is branching useful?
Branching is useful because it allows developers to work on different features or bug fixes in isolation from the main codebase. This means that changes can be made without affecting the stable version of the project, enabling experimentation and parallel development. Branching also facilitates collaboration, as multiple team members can work on separate branches simultaneously and merge their changes back into the main branch when ready.
